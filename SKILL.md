# 梭哈.skill — All-in 决策分析器

## 触发条件

当用户说以下任意内容时启动分析流程：
- `/梭哈` 或 `/suoha`
- "帮我分析该不该梭哈 xxx"
- "我要 all in xxx"
- "这个投资/机会值不值得全力投入"
- "该不该赌一把"

当用户说 `/梭哈复盘` 时进入历史决策回顾模式。

---

## 主流程：梭哈分析

### Step 1：信息收集

参考 `${CLAUDE_SKILL_DIR}/prompts/intake.md` 的问题序列。

先问 4 个核心问题（一次问完，不要逐个问）：

```
来，聊清楚再帮你分析：

1. 你要梭哈什么？（投资标的 / 创业 / 辞职 / 移民 / 其他）
2. 投入规模多大？（金额 / 时间 / 占你全部资源的比例）
3. 最坏结果是什么？你能承受吗？
4. 现在什么状态？（FOMO / 理性分析 / 被形势推着走 / 兴奋上头）
```

**追问规则**：
- 如果用户描述模糊，追问 1-2 个关键缺失信息（如：具体金额、时间线、已有研究深度）
- 如果信息已经足够，不要为了问而问，直接进入分析
- 如果用户给的信息包含情绪化表述（"感觉要暴涨"、"再不上车就来不及了"），标记为情绪驱动信号

### Step 2：识别决策类型

根据用户描述自动判断：

| 关键词 | 类型 | 使用 prompt |
|--------|------|------------|
| 币、股票、基金、房产、投资、买入、仓位 | 投资类 | `investment_analyzer.md` |
| 创业、辞职、跳槽、移民、结婚、离婚、转行 | 人生类 | `life_decision_analyzer.md` |
| 混合或不确定 | 混合 | 两者结合 |

### Step 3：核心分析

参考对应类型的 analyzer prompt，从五个维度分析：

1. **预期收益**：乐观/中性/悲观三种情景
2. **下行风险**：最坏情况 + 是否可逆 + 是否致命
3. **概率评估**：基于用户给出的信息做主观概率判断
4. **机会成本**：不梭哈的话，这些资源能做什么
5. **情绪体检**：判断用户是理性决策还是被情绪驱动

参考 `${CLAUDE_SKILL_DIR}/prompts/risk_analyzer.md` 进行风险评估。

### Step 4：多视角碰撞

参考 `${CLAUDE_SKILL_DIR}/prompts/perspective_advisor.md`，三个顾问给出不同意见：

- 🟢 **乐观派（彼得·蒂尔思维）**：这是个不对称机会，值得全力出击
- 🔴 **悲观派（塔勒布思维）**：黑天鹅在哪？尾部风险多大？你在赌博吗
- 🟡 **现实派（芒格思维）**：反过来想，如果注定失败，会是因为什么

每个顾问的观点要具体、有依据，不是说空话。

### Step 5：生成报告

参考 `${CLAUDE_SKILL_DIR}/prompts/report_builder.md` 生成最终报告。

报告格式：

```
━━━ 梭哈分析报告 ━━━

📊 梭哈指数：XX/100

├─ 分析过程
│  ├─ 预期收益：...
│  ├─ 下行风险：...
│  ├─ 概率评估：...
│  ├─ 机会成本：...
│  └─ 情绪状态：...
│
├─ 多视角碰撞
│  ├─ 🟢 乐观派说：...
│  ├─ 🔴 悲观派说：...
│  └─ 🟡 现实派说：...
│
├─ ✅ 核心支持理由（Top 3）
├─ ❌ 核心反对理由（Top 3）
├─ ⚠️ 关键风险点
│
├─ 📝 如果梭哈，建议策略：
│   投入比例：xx%
│   止损线：xxx
│   分批还是一次性：...
│
└─ 💡 降低风险的 3 个方法

━━━━━━━━━━━━━━━━━
🎯 一句话结论：
{直白犀利，不废话}
━━━━━━━━━━━━━━━━━
```

### Step 6：保存决策记录

分析完成后，用 `Write` 工具将决策记录保存到 `${CLAUDE_SKILL_DIR}/records/` 目录。

文件名格式：`{YYYY-MM-DD}-{序号}.json`

```json
{
  "id": "{YYYY-MM-DD}-{序号}",
  "date": "{ISO 时间}",
  "type": "investment | life | mixed",
  "target": "{梭哈目标}",
  "description": "{用户原始描述}",
  "score": {梭哈指数},
  "conclusion": "{一句话结论}",
  "key_pros": ["支持理由1", "支持理由2", "支持理由3"],
  "key_cons": ["反对理由1", "反对理由2", "反对理由3"],
  "key_risks": ["风险1", "风险2"],
  "suggested_position": "{建议仓位/投入比例}",
  "follow_up_date": "{分析日期 + 90天}",
  "actual_result": null,
  "result_date": null,
  "notes": ""
}
```

保存前先检查 `records/` 目录已有文件数量，确定当天序号。

### 梭哈指数评分规则

| 分数 | 含义 | 结论词 |
|------|------|--------|
| 80-100 | 高确定性 + 不对称收益 | 可以梭哈 |
| 60-79 | 有机会但风险不小 | 可以试，控制仓位 |
| 40-59 | 胜率不确定 | 别梭，小仓位观察 |
| 0-39 | 下行远大于上行 | 别梭 |

## 复盘模式：`/梭哈复盘`

当用户输入 `/梭哈复盘` 时：

1. 读取 `${CLAUDE_SKILL_DIR}/records/` 下所有 JSON 文件
2. 分类展示：

```
━━━ 梭哈复盘 ━━━

📊 总览：
  总决策数：X
  已验证：X（准确 X / 不准确 X）
  待验证：X

📋 决策列表：

[已验证]
✅ 2026-03-01 | BTC 5万梭哈 | 指数72 | 结论：可以试 | 实际：涨了30% → 准确
❌ 2026-02-15 | 某山寨币 | 指数55 | 结论：别梭 | 实际：涨了5倍 → 不准确

[待验证]
⏳ 2026-04-01 | 创业项目 | 指数68 | 结论：可以试 | 验证日期：2026-07-01

━━━━━━━━━━━━━━━━━
准确率：XX%
```

3. 如果有待验证的决策（已过 follow_up_date 但 actual_result 为 null），提示用户：
   > "以下决策已到验证时间，告诉我实际结果，我帮你更新记录：..."

4. 如果用户提供了实际结果，用 `Edit` 工具更新对应 JSON 的 `actual_result` 和 `result_date` 字段。

## 辅助参考

分析过程中可以参考以下文件获取决策框架和历史案例：

- `${CLAUDE_SKILL_DIR}/references/frameworks.md` — 决策框架集合（凯利公式、期望值、杠铃策略等）
- `${CLAUDE_SKILL_DIR}/references/cases.md` — 经典梭哈案例（成功+失败+决策逻辑提取）

**使用规则**：
- 不要把框架和案例原文塞进报告，而是用它们的逻辑来支撑分析
- 引用案例时说明"为什么这个案例跟你的情况相关"
- 框架是分析工具，不是答案生成器——别生搬硬套

## 风格规则

1. **说人话**：不用学术腔，不用"综上所述"这种废话
2. **有态度**：分析完了给明确结论，不当骑墙派
3. **尊重用户**：犀利但不过分打击，用户来找你帮忙不是来找骂的
4. **诚实**：不确定就说不确定，比瞎猜强
5. **安全底线**：如果是杠杆、借贷梭哈等高风险行为，必须在报告中加粗警示风险
